# MVP 등록 가이드 — 사용법

> **위치**: `/admin/settings`  
> **목적**: MVP 프로젝트 완성 시 플랫폼 등록 절차를 빠짐없이 수행하기 위한 인터랙티브 체크리스트

---

```
┌─────────────────────────────────────────┐
│  MVP 등록 가이드                         │
│  MVP 완성 시 플랫폼에 등록하는 표준...    │
├─────────────────────────────────────────┤
│  전체 진행률      ████████░░  8 / 17    │

├─────────────────────────────────────────┤

│  ...                                    │

```

## 사용 방법

### 기본 사용

1. `/admin/settings` 접속
2. 새 MVP가 완성되면 **STEP 1부터 순서대로** 각 항목을 클릭해 체크
3. 체크된 항목은 취소선 + 초록 아이콘으로 표시
4. 상단 진행률 바가 실시간 업데이트
5. 모든 항목 완료 시 "✓ 완료" 표시

### 초기화

- 화면 하단 "체크 초기화" 클릭 → 새로운 MVP 작업 시작 시 사용

> **참고**: 체크 상태는 브라우저 메모리에만 유지됩니다. 페이지 새로고침 시 초기화됩니다.

---

## 4단계 워크플로우 상세

### STEP 1 — AdminProjects 등록

`/admin/projects` → 상단 "새 프로젝트" 버튼

| 필드 | 작성 요령 |
|------|-----------|
| **slug** | 영문 소문자 + 하이픈만 사용. 예: `baikal-rag-search` |
| **title** | 기능 중심 명사형. 예: "BAIKAL RAG 검색 시스템" |
| **subtitle** | 한 문장 요약. 예: "기업 내부 문서를 AI로 검색하는 RAG 파이프라인" |
| **description** | 마크다운 형식. `## 개요` / `## 주요 기능` / `## 성과` 구조 권장 |
| **category** | DB 값과 정확히 일치해야 필터 동작: `NLP`, `AI Agent`, `Platform / Infra`, `Research / Experiment`, `Computer Vision` |
| **tech_stack** | 콤마 구분 입력. 예: `Python, FastAPI, OpenAI` |
| **featured** | 메인 홈 Featured 섹션 노출 여부. 대표작 4~6개만 선별 권장 |
| **status** | `active` = 공개, `planning` = 개발 예정, `archived` = 보관 |

**description 마크다운 템플릿:**
```markdown
## 개요

한두 문장으로 프로젝트 배경과 핵심 정의.

## 주요 기능

- 기능 1
- 기능 2
- 기능 3

## 기술 스택

사용한 핵심 기술과 아키텍처 선택 이유.

## 성과 / 현황

- 측정 가능한 수치 (정확도 X%, 응답시간 Xs 등)
- 현재 운영 상태
```

---

### STEP 2 — AdminContents 콘텐츠 3종 등록

`/admin/contents` → "새 콘텐츠" 버튼

MVP 1개당 최소 **2~3개 콘텐츠** 등록을 권장합니다. 각 타입의 역할이 다릅니다.

#### article — SEO 핵심 자산
- **제목 패턴**: `[프로젝트명] 아키텍처와 구현 과정`
- **길이**: 1,000자 이상 (SEO 최소 기준)
- **포함 내용**: 기술 선택 이유, 코드 예시, 성능 수치
- `is_featured: TRUE` 설정 권장 → 메인 Featured 섹션 노출

#### note — 개발일지 / 개인 브랜딩
- **제목 패턴**: `[개발일지 #N] [프로젝트명]을 만들면서 배운 것`
- **길이**: 300~600자 (짧고 읽기 쉽게)
- **포함 내용**: 실패 경험, 예상치 못한 문제, 깨달음

#### research — 실험 데이터 (해당 시)
- **제목 패턴**: `[A vs B] 비교 실험: [측정 항목]`
- **포함 내용**: 실험 조건, 측정값 표, 결론
- 수치 데이터가 없으면 생략 가능

**공통 주의사항:**
- `status: published`로 저장해야 공개 페이지에 노출
- `slug`는 영문, 하이픈, 프로젝트명과 연관된 키워드 포함 권장

---

### STEP 3 — 외부 채널 배포

콘텐츠를 등록한 후 외부로 배포해 트래픽을 유입시킵니다.

| 채널 | 형식 | 링크 포함 여부 |
|------|------|---------------|
| **X (Twitter)** | 핵심 인사이트 1가지 + 사이트 URL | 필수 |
| **LinkedIn** | 기술 배경 + article 요약 또는 풀 본문 | 필수 |
| **커뮤니티** | 개발자 슬랙, 오픈 커뮤니티 등 | 선택 |

> X 포스팅은 RSS 자동 수집(STEP 4)으로 플랫폼에 역유입되므로 꼭 포스팅.

---

### STEP 4 — RSS 소스 등록으로 역유입 자동화

외부 채널 글이 플랫폼의 `x_post` 콘텐츠로 자동 집결되도록 설정합니다.

1. `/admin/sources` → 채널 RSS URL 등록
2. `/admin/sync-jobs` → 싱크 실행 → 수집 결과 확인
3. `/admin/contents` → 수집된 x_post 중 불필요한 항목 `status: draft`로 숨김

| 소스 타입 | RSS/수집 방식 |
|-----------|--------------|
| X (Twitter) | RSS Bridge 또는 Nitter RSS |
| 개인 블로그 | 블로그 RSS 피드 URL |
| YouTube | 채널 RSS URL |

## 등록 완료 기준

아래 4가지를 충족하면 MVP 등록 완료입니다.

- [ ] `/projects/{slug}` 상세 페이지 정상 렌더링 확인
- [ ] `/content` 목록에 관련 article 1건 이상 노출 확인
- [ ] 홈 화면 Featured 섹션에 프로젝트 또는 article 노출 확인 (featured=true 인 경우)
- [ ] X 포스팅 후 싱크 시 x_post 자동 수집 확인

## 관련 어드민 페이지

| 페이지 | 경로 | 역할 |
|--------|------|------|
| 프로젝트 관리 | `/admin/projects` | 프로젝트 CRUD |
| 콘텐츠 관리 | `/admin/contents` | 콘텐츠 CRUD |
| 소스 관리 | `/admin/sources` | RSS 소스 등록 |
| 싱크 관리 | `/admin/sync-jobs` | 자동 수집 실행 및 확인 |
| **MVP 등록 가이드** | `/admin/settings` | **이 체크리스트** |
