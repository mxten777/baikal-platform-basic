# 개발 작업 일지 — 2026-03-14

## 오늘 해결한 문제

### 1. RSS 콘텐츠가 사이트에 표시되지 않는 문제

**원인**: `sync-rss` Edge Function이 수집한 콘텐츠를 `status: 'pending'`으로 저장했으나
프론트엔드 `getContents()`는 `.eq('status', 'published')` 만 조회함

**해결**: `supabase/functions/sync-rss/index.ts` 수정
```diff
- status: 'pending',
+ status: 'published',
```
Edge Function 재배포: `supabase functions deploy sync-rss`

---

### 2. `rss_sources` 테이블이 비어있는 문제

**원인**: 마이그레이션 시 `rss_sources` 테이블에 데이터가 입력되지 않아 동기화 대상이 없었음

**해결**: SQL Editor에서 수동 삽입
```sql
INSERT INTO rss_sources (source_id, feed_url, feed_type, site_name, language)
VALUES (
  'ee63c773-299a-4262-aedc-2954d57e3eb1',
  'https://rss.blog.naver.com/jngdy',
  'rss2',
  '네이버 블로그 - jngdy',
  'ko'
);
```

---

### 3. etag 캐시로 인해 새 포스트가 수집되지 않는 문제

**원인**: `rss_sources.last_etag` / `last_modified` 값이 남아있어 서버가 304 Not Modified 응답

**해결**: SQL Editor에서 캐시 초기화 후 재동기화
```sql
UPDATE rss_sources SET last_etag = NULL, last_modified = NULL;
```

---

### 4. `rss_sources` 중복 행 제거

**원인**: 동기화 테스트 과정에서 `site_name = NULL`인 중복 행이 생성됨

**해결**:
```sql
DELETE FROM rss_sources WHERE site_name IS NULL;
```

---

### 5. 크론잡 미등록 문제

**원인**: `cron_setup.sql`이 실행되지 않았고 `pg_cron`, `pg_net` Extension도 비활성화 상태

**해결**:
1. Supabase 대시보드 → Database → Extensions → `pg_cron` 활성화
2. Supabase 대시보드 → Database → Extensions → `pg_net` 활성화
3. SQL Editor에서 크론잡 등록 (매 30분 RSS 자동 동기화)
